Process Icons — Which style works for you?

Terminal icons are real and widely supported. Three approaches below.

A

Nerd Font Icons

Crisp, developer-grade icons. Requires a Nerd Font (FiraCode, Meslo, JetBrains Mono NF, etc.) — most devs already have one. Falls back gracefully if not installed.

Pros

  • Looks amazing in Nerd Font terminals
  • Per-process color coding
  • Used by eza, lazygit, etc.

Cons

  • Requires Nerd Font installed
  • Falls back to plain text otherwise
B

Emoji Icons

Works everywhere — no special font needed. Every modern terminal renders emoji natively.

  • Less crisp than Nerd Fonts
  • Spacing can be inconsistent across terminals
C

Smart Auto-detect

Detect if terminal supports Nerd Fonts → use them. Otherwise fall back to emoji. User can override with --icons=nerdfont|emoji|none.

portly  ✓ nerd font detected
Auto-selects best icon style.
Falls back gracefully.
portly --icons=none for plain text.

Pros

  • Best experience on any setup
  • No config needed by default
  • Escape hatch for plain text

Cons

  • Slightly more complex to implement
